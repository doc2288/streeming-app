## 2024-05-15 - AuthModal Form Validation and Accessibility
**Learning:** Adding `required` and `minLength` directly onto form `<input>` components prevents the custom `.form-error` message from showing when using browser native HTML5 validation, unless `noValidate` is explicitly set on the `<form>`. However, linking explicit `htmlFor`/`id` on labels, native HTML5 validations, and `role="alert"` on error text correctly covers multiple accessibility/UX fronts (screen reader context, immediate browser-level input hints, and dynamic live-region announcements for application errors).
**Action:** Always combine semantic label bindings (`htmlFor`/`id`), native HTML5 validation constraints, and `role="alert"` error containers when writing standard forms, ensuring that screen readers properly announce context and feedback.
